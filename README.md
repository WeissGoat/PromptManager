# AI 提示词库管理工具 (Prompt Manager)

一个基于 PySide6 (Qt) 开发的图形化 AI 提示词库管理工具，专为管理和组织大量提示词场景和动作节点而设计。

## ✨ 功能特性

### 🎯 核心功能

- **场景与节点管理**
  - 树形结构管理多个场景（Scenes）
  - 每个场景包含多个动作节点（Action Nodes）
  - 支持节点拖拽排序和自动编号
  - 支持跨场景链接节点（通过 Windows 快捷方式）

- **提示词编辑**
  - 实时编辑和保存提示词内容
  - 支持多行文本编辑
  - 内置搜索功能（Ctrl+F）
  - 差异对比显示（与上一次选中的节点对比）

- **图片预览系统**
  - 自动检测文件夹中的图片
  - 支持多种图片格式（PNG, JPG, JPEG, WEBP, BMP）
  - 鼠标滚轮切换图片
  - 左键切换文件夹，右键返回
  - 智能图片缩放和优化加载

- **批量操作**
  - 批量导出选中节点的提示词
  - 批量编辑和追加提示词
  - 支持多选操作

- **书签系统**
  - 为常用场景添加书签标记
  - 书签场景高亮显示
  - 书签数据持久化保存

- **差异统计**
  - 自动计算相邻节点间的标签差异
  - 可视化显示新增（+）和删除（-）的标签数量
  - 异步计算，不阻塞 UI

- **脚本运行**
  - 支持运行外部批处理脚本
  - 可配置运行参数（Key-Value 格式）
  - 参数配置持久化保存

### 🛠️ 高级功能

- **Windows 快捷方式支持**
  - 支持 `.lnk` 文件作为节点
  - 自动解析快捷方式目标路径
  - 可创建跨场景的节点链接

- **右键菜单**
  - 场景菜单：书签、重命名、在资源管理器中打开、发送到
  - 节点菜单：重命名、删除（移至回收站）、重置排序

- **智能排序**
  - 自然排序（1, 2, 10 而非 1, 10, 2）
  - 支持自动编号格式：`1_Name` 或 `(1)Name`
  - 可重置所有序号并重新排序

## 📋 系统要求

- **Python**: >= 3.11
- **操作系统**: Windows (主要支持), Linux/macOS (部分功能可能受限)
- **依赖库**:
  - PySide6 >= 6.10.1
  - pywin32 >= 311 (Windows 快捷方式支持，可选)

## 🚀 快速开始

### 1. 克隆仓库

```bash
git clone https://github.com/yourusername/PromptManager.git
cd PromptManager
```

### 2. 安装依赖

**推荐使用 [uv](https://github.com/astral-sh/uv)（快速且现代）:**

```bash
# 安装 uv（如果尚未安装）
# Windows (PowerShell)
irm https://astral.sh/uv/install.ps1 | iex

# macOS/Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

# 使用 uv 安装依赖并运行
uv sync
uv run python PromptManager.py
```


### 3. 配置路径（可选）

首次运行前，建议修改 `PromptManager.py` 中的默认路径：

```python
# 第 446 行：修改为你的根目录路径
self.root_dir = r"D:\AI\design\动作改2"

# 第 451 行：修改为你的默认脚本路径（可选）
self.bat_script_path = r"C:\Users\...\script.bat"
```

如果不修改，程序启动时会提示选择根目录。

### 4. 运行应用

**使用 uv:**

```bash
uv run python PromptManager.py
```

**或直接运行:**

```bash
python PromptManager.py
```

## 🎮 使用指南

### 基本操作

1. **选择根目录**
   - 启动应用后，选择包含场景文件夹的根目录
   - 或修改代码中的 `self.root_dir` 默认路径

2. **浏览场景**
   - 左侧树形视图显示所有场景
   - 点击场景名称加载该场景的节点列表

3. **管理节点**
   - 中间列表显示当前场景的所有动作节点
   - 点击节点查看和编辑提示词
   - 拖拽节点可重新排序（自动重命名）

4. **编辑提示词**
   - 右侧编辑器显示当前节点的 `tags.txt` 内容
   - 编辑后点击"保存"按钮保存更改
   - 使用 `Ctrl+F` 打开搜索功能

5. **预览图片**
   - 预览区域自动显示节点文件夹中的图片
   - **左键点击**: 切换到下一个包含图片的文件夹
   - **右键点击**: 返回上一个文件夹
   - **鼠标滚轮**: 在当前文件夹中切换图片

### 高级操作

#### 创建新场景

1. 点击"新建组合场景"按钮
2. 输入场景名称
3. 从其他场景拖拽节点到新场景（创建链接）

#### 批量导出

1. 在节点列表中多选节点（Ctrl+点击）
2. 点击"批量导出"按钮
3. 提示词内容将合并并显示在编辑器中
4. 同时创建临时文件，可复制到剪贴板

#### 批量编辑

1. 选择多个节点
2. 点击"批量编辑"按钮
3. 输入要追加的提示词内容
4. 系统会自动更新所有选中节点

#### 运行脚本

1. 选择要处理的节点（可选，默认使用当前场景）
2. 点击"运行"按钮
3. 首次运行需要选择 `.bat` 脚本文件
4. 在参数对话框中配置运行参数
5. 点击"运行"执行脚本

#### 使用书签

1. 右键点击场景
2. 选择"加入书签"或"取消书签"
3. 书签场景会显示 ⭐ 标记并高亮显示

## 🎨 界面说明

### 三栏布局

- **左侧**: 场景树形列表
- **中间**: 动作节点列表（带差异统计）
- **右侧**: 
  - 图片预览区域
  - 提示词编辑器
  - 差异对比视图
  - 操作按钮

### 差异统计显示

节点列表中每个节点右侧会显示：
- **绿色标签 (+N)**: 相比前一个节点新增的标签数量
- **红色标签 (-N)**: 相比前一个节点删除的标签数量

## 🔧 配置说明

### 目录结构要求

应用期望的目录结构如下：

```
根目录/
├── 场景1/
│   ├── 1_动作节点1/
│   │   ├── tags.txt          # 提示词文件（必需）
│   │   └── *.png/jpg          # 参考图片（可选）
│   ├── 2_动作节点2/
│   │   └── tags.txt
│   └── 3_动作节点3.lnk        # 快捷方式链接（可选）
├── 场景2/
│   └── ...
└── ...
```

**重要**: 每个节点文件夹必须包含 `tags.txt` 文件才能被识别为有效节点。

### 默认路径配置

在 `PromptManager.py` 中可以修改以下默认路径：

```python
# 第 446 行：根目录路径
self.root_dir = r"D:\AI\design\动作改2"

# 第 451 行：默认批处理脚本路径（可选）
self.bat_script_path = r"C:\Users\...\script.bat"
```

### 数据文件

应用会自动创建以下配置文件：

- `bookmarks.json`: 存储书签信息（自动生成）
- `run_params.json`: 存储运行参数配置（自动生成）

这些文件位于应用主目录，与 `PromptManager.py` 同目录。

## 🐛 故障排除

### Windows 快捷方式支持

如果遇到 `.lnk` 文件无法解析的问题：

```bash
pip install pywin32
```

### 图片无法显示

- 确保图片文件格式受支持（PNG, JPG, JPEG, WEBP, BMP）
- 检查文件路径是否包含特殊字符
- 确认文件权限正常

### 节点排序问题

- 如果节点编号混乱，可以使用"重置排序"功能
- 支持从旧格式 `(1)Name` 自动迁移到新格式 `1_Name`

## 💡 使用技巧

### 快捷键

- `Ctrl+F`: 在提示词编辑器中打开搜索
- `Enter`: 在搜索框中查找下一个匹配项
- `Shift+Enter`: 在搜索框中查找上一个匹配项

### 节点命名规范

- **推荐格式**: `1_Name`, `2_Name`, `3_Name` ...
- **旧格式支持**: `(1)Name`, `(2)Name` ... (会自动迁移)
- 拖拽排序后会自动重新编号

### 性能优化

- 差异计算在后台线程进行，不会阻塞 UI
- 图片预览使用懒加载，只加载可见图片
- 使用 `os.scandir()` 优化大目录扫描性能
